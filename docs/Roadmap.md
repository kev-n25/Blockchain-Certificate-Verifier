# Development Roadmap

# Blockchain-Based Academic Certificate Verification System using ShaktiDB

---

## Project Duration

**Planned Duration:** 4 Weeks

The project is divided into four phases, each focusing on a specific aspect of the system development.

---

# Week 1 – Planning and System Design

## Objectives

* Define project scope and objectives
* Perform literature review
* Analyze existing certificate verification systems
* Design overall system architecture
* Design database schema
* Prepare GitHub repository and documentation

### Deliverables

* Project proposal
* System architecture
* Database design
* API design
* Development roadmap
* Initial GitHub repository

---

# Week 2 – Backend Development and Database Integration

## Objectives

* Set up the Flask backend
* Configure the development environment
* Integrate ShaktiDB
* Develop database models
* Implement administrator authentication
* Develop certificate upload functionality

### Deliverables

* Functional backend
* Database connection
* Certificate upload module
* User authentication

---

# Week 3 – Blockchain and Certificate Verification

## Objectives

* Implement the blockchain module
* Generate SHA-256 hashes
* Store certificate hashes in blockchain blocks
* Develop certificate verification functionality
* Implement verification logs

### Deliverables

* Blockchain implementation
* Hash generation module
* Certificate verification module
* Verification history

---

# Week 4 – QR Code Integration, Testing and Deployment

## Objectives

* Generate QR codes for certificates
* Implement QR-based verification
* Improve user interface
* Perform application testing
* Fix bugs and optimize performance
* Complete documentation
* Prepare final project demonstration

### Deliverables

* QR code verification
* Tested application
* Updated documentation
* Final GitHub repository
* Project presentation

---

# Expected Final Features

* Administrator login
* Secure certificate upload
* Student certificate management
* ShaktiDB integration
* Blockchain ledger
* SHA-256 certificate hashing
* Certificate verification
* QR code generation
* Verification history
* Tamper detection
* User-friendly web interface

---

# Project Milestones

| Week   | Milestone                                                              |
| ------ | ---------------------------------------------------------------------- |
| Week 1 | Planning and system design completed                                   |
| Week 2 | Backend and database integration completed                             |
| Week 3 | Blockchain and verification modules completed                          |
| Week 4 | Testing, QR integration, documentation, and final deployment completed |

---

# Success Criteria

The project will be considered successful if it:

* Securely stores certificate metadata using ShaktiDB.
* Detects certificate tampering through blockchain-based hash verification.
* Enables fast and reliable certificate verification.
* Provides an intuitive web interface for administrators and verifiers.
* Demonstrates the practical application of blockchain technology in academic certificate verification.
